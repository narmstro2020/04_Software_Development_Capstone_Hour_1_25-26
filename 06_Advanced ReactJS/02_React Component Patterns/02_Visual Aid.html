React Component Patterns - Visual Guide

Container-Presentational Pattern

Container (Smart)
Presentational (Dumb)
UserContainer
Smart Component
  • Manages state
  • Fetches data
  • Contains logic
  • Uses hooks
Props
UserDisplay
Dumb Component
  • Receives props
  • Renders UI
  • No business logic
  • Reusable
Example Code Structure:
// Container Component const DataContainer = () => { const [data, setData] = useState([]); useEffect(() => { // Fetch data logic fetchData().then(setData); }, []); return <DataDisplay data={data} />; } // Presentational Component const DataDisplay = ({ data }) => { return ( <div> {data.map(item => <div>{item}</div>)} </div> ); }

List-Item Pattern

List Component
Item Component
TodoList
List Component
  • Maps over data
  • Handles collection
  • Passes props to items
Iterates
TodoItem
Item Component
  • Renders single item
  • Handles item events
× Multiple Items
Example Implementation:
// List Component const TodoList = ({ todos, onToggle }) => { return ( <ul> {todos.map(todo => ( <TodoItem key={todo.id} todo={todo} onToggle={onToggle} /> ))} </ul> ); } // Item Component const TodoItem = ({ todo, onToggle }) => { return ( <li onClick={() => onToggle(todo.id)}> {todo.text} </li> ); }

Key Principles

Single Responsibility
Each component does one thing well
Props Down, Events Up
Data flows down, actions bubble up
Reusability
Components work with different data
Testability
Components can be tested in isolation
