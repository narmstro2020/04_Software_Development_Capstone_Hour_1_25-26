CSS Styling Methods in React

Inline Styles

const buttonStyle = { backgroundColor: '#007bff', color: 'white', padding: '10px 20px' }; return <button style={buttonStyle}>

✅ Pros

  • Dynamic styling with props/state
  • Component-scoped by default
  • No external files needed
  • Direct JavaScript integration

❌ Cons

  • No pseudo-classes (:hover, :focus)
  • No media queries
  • Can clutter JSX
  • Performance impact with complex styles

Best For:

Dynamic styling based on component state, quick prototyping, simple animations

External CSS Files

/* Button.css */ .primary-button { background-color: #007bff; color: white; } // Button.jsx import './Button.css';

✅ Pros

  • Full CSS features (hover, media queries)
  • Familiar CSS syntax
  • Better performance
  • Easier for designers

❌ Cons

  • Global namespace pollution
  • Class name conflicts
  • Harder to maintain
  • Less dynamic capabilities

Best For:

Complex layouts, global styles, teams familiar with traditional CSS, responsive design

CSS Modules

/* Button.module.css */ .primary { background-color: #007bff; } // Button.jsx import styles from './Button.module.css';

✅ Pros

  • Locally scoped styles
  • No naming conflicts
  • Explicit dependencies
  • Full CSS features

❌ Cons

  • Build step required
  • Learning curve
  • Verbose class references
  • Debugging can be harder

Best For:

Medium to large projects, component libraries, teams wanting style isolation

Styled Components

const Button = styled.button` background-color: ${props => props.primary ? '#007bff' : 'gray'}; color: white; `;

✅ Pros

  • Component-based styling
  • Dynamic styling with props
  • Automatic vendor prefixing
  • Theme support

❌ Cons

  • Additional library dependency
  • Runtime overhead
  • Learning curve
  • Bundle size increase

Best For:

Component libraries, heavy prop-based styling, design systems, theme-heavy applications

Decision Matrix

Quick & Dynamic: Choose Inline Styles
Traditional & Global: Choose CSS Files
Scoped & Maintainable: Choose CSS Modules
Component-Driven: Choose Styled Components
